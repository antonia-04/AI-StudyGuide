

În cadrul **algoritmilor evolutivi (AE)**, alegerea unei **reprezentări adecvate** pentru o soluție candidat este esențială. Această alegere influențează tipul de codificare folosit (genotip), precum și cum se aplică operatorii de selecție, recombinare și mutație.

Mai jos sunt prezentate diferite tipuri de probleme care pot fi rezolvate cu AE, alături de **reprezentarea** și **funcția de fitness** asociate.


#### ■ Problema **rucsacului** (Knapsack Problem)

- **Context**: Avem un rucsac de capacitate fixă și un set de obiecte fiecare cu greutate și valoare. Scopul este să alegem un subset de obiecte astfel încât **să maximizăm valoarea totală**, fără a depăși capacitatea rucsacului.
    
- **Reprezentare**: Liniară discretă binară.  
    Fie un vector binar $x = [x_1, x_2, ..., x_n]$, unde:
    - $x_i = 1$ înseamnă că obiectul $i$ este inclus,
    - $x_i = 0$ altfel.
- **Fitness**:  
    Se penalizează diferența dintre capacitatea rucsacului și suma greutăților obiectelor selectate:
    $f(x) = \left| \text{capacitate} - \sum_{i=1}^{n} x_i \cdot \text{greutate}_i \right|$
    
    Scopul este **minimizarea** acestei diferențe (sau maximizarea valorii cu constrângere).
- **Legătura cu AE**: Se folosesc operatori genetici pe vectori binari (mutarea unui `1` în `0` sau invers). Soluțiile evoluează înspre o selecție optimă de obiecte.


#### ■ Problema **plății unei sume** folosind monede

- **Context**: Dorim să compunem o sumă fixă de bani utilizând un set de monede de diferite valori, într-un număr cât mai apropiat de suma dorită.
    
- **Reprezentare**: Liniară discretă întreagă.  
    Fie un vector $x = [x_1, x_2, ..., x_n]$ unde:
    - $x_i$ reprezintă numărul de monede de tipul $i$ selectate.
- **Fitness**:  
    Se evaluează diferența absolută între suma generată și suma țintă:
    $f(x) = \left| \text{suma dorită} - \sum_{i=1}^{n} x_i \cdot \text{valoare}_i \right|$
    
    Se urmărește **minimizarea** acestei diferențe.
- **Legătura cu AE**: Se aplică mutații discrete (incrementări/decrementări) și selecție pe bază de cât de aproape este suma compusă față de suma dorită.
    

#### ■ Problema **comisului voiajor** (Travelling Salesman Problem – TSP)

- **Context**: Se dorește găsirea celui mai scurt traseu care trece o singură dată prin fiecare oraș și se întoarce la punctul de plecare.
- **Reprezentare**: Liniară discretă întreagă – **permutare**.  
    Fie un vector $x = [x_1, x_2, ..., x_n]$ care conține o permutare a orașelor.
- **Fitness**:  
    Costul total (lungimea traseului):
    $f(x) = \sum_{i=1}^{n-1} \text{dist}(x_i, x_{i+1}) + \text{dist}(x_n, x_1)$
    
    Scopul este **minimizarea** costului total al rutei.
- **Legătura cu AE**: Se folosesc operatori de recombinare pentru permutări (ex: Partially Mapped Crossover) și mutații specifice (ex: inversarea unei subsecvențe).


#### ■ Problema **optimizării funcțiilor reale**

- **Context**: Dorim să găsim valoarea $x$ care minimizează sau maximizează o funcție matematică $f(x)$, unde $x$ poate fi un vector cu valori reale.
- **Reprezentare**: Liniară continuă – vector real.  
    $x = [x_1, x_2, ..., x_n]$, fiecare $x_i \in \mathbb{R}$
- **Fitness**:  
    Direct valoarea funcției:
    $f(x) = \text{valoarea funcției în punctul } x$
    
    Obiectivul este **minimizare** sau **maximizare** în funcție de problemă.
- **Legătura cu AE**: Se folosesc operatori de mutație reală (ex: adăugare de zgomot gaussian) și selecție bazată pe performanță.

#### ■ Problema **calculului ariei unui cerc** (simbolic)

- **Context**: Dorim ca algoritmul să **descopere** o formulă simbolică (de exemplu, $f(r) = \pi r^2$) care să aproximeze aria unui cerc pe baza unor exemple.
- **Reprezentare**: **Arborescentă** – copaci de expresii simbolice.  
    Fiecare individ este o structură de tip arbore care combină constante, variabile și operații.
- **Fitness**:  
    Suma pătratelor erorilor pe un set de exemple $(r_i, \text{aria reală}_i)$:
    $f(x) = \sum_{i=1}^{m} \left( \text{aria reală}_i - f(r_i) \right)^2$
    
    Se urmărește **minimizarea** erorii.
- **Legătura cu AE**: Se aplică **Programare Genetică (GP)**, în care cromozomii sunt arbori și operatorii lucrează cu subarbori (recombinare, mutație de expresii etc.).
