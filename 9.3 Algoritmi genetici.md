
Sunt o **subclasă** de algoritmi evolutivi.

- Folosesc tipic **codificare binară** sau vectori de numere.
- Folosesc **crossover (recombinare)** ca operator principal.
- Selectează indivizii în funcție de **fitness**.
- Sunt foarte utilizați pentru probleme combinatorii sau de optimizare numerică.

Algoritmii genetici sunt metode de **optimizare** inspirate din **selecția naturală** și **evoluția biologică**. Ei simulează procesul de evoluție al unei populații de indivizi pentru a găsi soluții optime sau aproape optime la probleme complexe.


#### Componentele de bază ale unui algoritm genetic

1. **Reprezentarea soluțiilor** (codificarea)
2. **Populația inițială**
3. **Funcția de fitness**
4. **Selecția**
5. **Recombinarea (crossover)**
6. **Mutația**
7. **Evaluarea**
8. **Criteriul de oprire**


####  Reprezentarea soluțiilor

Orice soluție este reprezentată sub forma unui **cromozom** (genotip), de obicei un **vector unidimensional** (de tip boolean, numeric, string etc.).

Exemplu:

- Probleme de tip binar: cromozom de forma  
    $x = [1, 0, 0, 1, 1, 0, 1]$
    
- Probleme de tip real:  
    $x = [2.3, -1.5, 0.7, 4.0]$
    

Cromozomul este transformat într-o soluție interpretabilă (fenotip), care poate fi evaluată într-un anumit **spațiu al problemei**.


#### Niveluri de existență ale unei soluții

- **Genotip**: Reprezentarea internă (cromozom)
- **Fenotip**: Interpretarea externă (individul în contextul problemei)

####  Inițializarea populației

Se generează o populație inițială de $N$ indivizi (cromozomi), de obicei aleator.

$P^{(0)} = \{x_1, x_2, ..., x_N\}$


#### Funcția de fitness

Fiecare individ este evaluat cu ajutorul unei **funcții de fitness** $f(x)$, care măsoară **calitatea** soluției.

- Poate fi funcția obiectivă directă sau derivată din ea.
- Valoarea fitness influențează șansa individului de a fi selectat.
    

#### Selecția

Selecția are rolul de a favoriza indivizii **mai buni** (cu fitness mare) pentru a participa la reproducere.

Metode comune:

- **Selecție proporțională (roulette wheel):** probabilitatea de selecție este proporțională cu fitnessul
- **Selecție prin turneu:** se aleg aleator $k$ indivizi și se selectează cel mai bun
- **Selecție elitistă:** cei mai buni indivizi trec automat mai departe

#### Crossover (recombinare)

Creează noi indivizi (descendenți) prin combinarea genelor a doi părinți.

Exemple:

- **One-point crossover**  
    Se alege un punct $k$ și se face schimb de gene:
    $\text{offspring}_1 = [x_1, ..., x_k, y_{k+1}, ..., y_n]$
- **Uniform crossover**  
    Se alege aleator pentru fiecare genă dacă vine de la părintele 1 sau 2.
    

#### Mutația

Se aplică cu o probabilitate mică pentru a introduce **diversitate** și a evita stagnarea.

Exemple:

- **Mutare bit** (pentru cromozomi binari): se inversează un bit
	$x_i \leftarrow 1 - x_i$
- **Perturbare mică** (pentru cromozomi reali): se adaugă zgomot gaussian
	$x_i \leftarrow x_i + \mathcal{N}(0, \sigma)$


#### Evaluare

Noii indivizi (după crossover și mutație) sunt evaluați folosind aceeași funcție de fitness $f(x)$.

#### Criterii de oprire

Algoritmul se oprește când:

- Se atinge un număr maxim de generații
- Nu se mai îmbunătățește fitnessul
- Se găsește o soluție satisfăcătoare

#### Pseudocod – Algoritm Genetic

```
1. Generează populația inițială P(0)
2. Pentru fiecare individ x din P(0): evaluează f(x)
3. Pentru t = 1 până la max_generations:
     a. Selectează părinți din P(t-1)
     b. Aplică crossover și mutație pentru a obține P'(t)
     c. Evaluează fiecare individ din P'(t)
     d. Selectează indivizi pentru P(t) (selecție + elitism)
4. Returnează cel mai bun individ
```
